# 프로젝트 목표: 견적서 PDF → CSV 자동 추출기 (quote-extractor)

## 개요
한국어 견적서 PDF 파일에서 물품 정보를 자동으로 추출하여 CSV(또는 엑셀)로 저장하는 도구.
AI 모델을 활용해 다양한 형식의 견적서를 유연하게 파싱한다.

## 파일 구조
```
quote-extractor/
├── config.json          ← 컬럼 정의, AI 프롬프트 설정
├── main.py              ← 진입점 (CLI)
├── pdf_reader.py        ← 1단계: PDF 텍스트 추출
├── ai_analyzer.py       ← 2단계: AI 분석 요청
├── column_mapper.py     ← 3단계: 컬럼 매핑 & 검증
├── csv_writer.py        ← 4단계: CSV 출력
└── output/              ← 결과 파일 저장 폴더
```

## 처리 단계

### 1단계: PDF 텍스트 추출 (`pdf_reader.py`)
- `pdfplumber`로 표(table) 구조 인식
- 이미지 기반 PDF(스캔본) 대비 OCR 옵션 지원

### 2단계: AI 분석 (`ai_analyzer.py`)
- 추출된 텍스트를 AI 모델에 전달
- 프롬프트 예시:
  > "아래는 견적서 PDF에서 추출한 텍스트야. 여기서 각 물품의 {물품명, 규격, 수량, 단가, 금액, 날짜}를 JSON 배열로 추출해줘. 해당 정보가 없으면 null로 남겨."
- 컬럼명 변형("Q'ty" 등)이나 숫자 포맷(콤마 포함) 등은 AI가 문맥으로 처리

### 3단계: 컬럼 매핑 & 검증 (`column_mapper.py`)
- AI 반환 JSON을 `config.json`에 정의된 컬럼 구조와 비교
- 빠진 항목은 빈 값으로 채우고, 불필요한 항목은 제외
- 누락 정보에 대한 피드백 제공 (예: "이 견적서에 날짜 정보가 없습니다")

### 4단계: CSV 출력 (`csv_writer.py`)
- 정리된 데이터를 CSV 또는 엑셀 파일로 저장
- 누적 모드: 여러 견적서를 하나의 CSV에 append

## 추가 기능 목표
- **설정 파일 분리** — `config.json`으로 컬럼 정의 외부화, 코드 수정 없이 컬럼 변경 가능
  ```json
  {"columns": ["물품명", "규격", "수량", "단가", "금액"], "date_field": true}
  ```
- **누적 모드** — 여러 PDF를 처리해도 하나의 CSV에 이어 붙이기 (append)
- **신뢰도 표시** — 각 추출 항목에 "확실함 / 추정값 / 정보없음" 태그 추가 → 검수 효율화
- **폴더 감시 자동화** — `watchdog` 라이브러리로 특정 폴더에 PDF 추가 시 자동 처리

## AI 모델
- **Google Gemini 2.0 Flash** (1순위): 무료 티어(분당 15회, 일 1,500회), PDF 직접 전달 가능, 한국어 이해도 우수
- **Ollama 로컬 모델** (2순위): 완전 무료, 오프라인 사용 가능, 추천 모델 — Qwen2.5 / EXAONE 3.0 / gemma3
  - 단, PDF 직접 처리 불가 → 텍스트 추출 후 전달 필요
